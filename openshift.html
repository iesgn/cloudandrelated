Traditional deployment

  • The applications are traditionally implemented as a single set of libraries and configuration files.
  • They are implemented in an operating system (on physical or virtual servers) with a set of services running (web, database, ...).
  • Disadvantage 1: O.S. updates can interrupt the application.
  • Disadvantage 2: If several applications are running on the same system, the library updates of one of them can affect other applications.
  • Possible solution: High availability systems that minimize the downtime of an application.

What alternatives can be used today?

Containers


Source: Red Hat

What are containers?

Containers are a type of partition isolated within a single operating system. They offer many of the same benefits as virtual machines, such as security, storage, and network isolation, but require far fewer hardware resources and are quicker to start and stop. They also isolate the libraries and runtime environment (such as CPU and storage) used by an application to minimize the impact of an OS upgrade on the host OS.

Containers benefits

  • Environment isolation
  • Smaller hardware resources needed
  • Quick deployment
  • Components reused
  • The impact of errors/changes can be minimized

Containers disadvantage

  • Complexity

Deployment of Microservice based applications

Containers promote the microservice based application development approach (vs monolithic applications), because the different services into which we the application is separated can easily run in different containers.

Advantages: Each container offers a service, they are easier to maintain, to scale, to start, to update,...

OpenShift

Development Platform, with Cloud Computing (PaaS) features developed by Red Hat

  • We focus on application development
  • Openshift uses Docker (CRI-O) and Kubernetes internally
  • It allows us to deploy applications in different environments (development, production, ...)
  • It facilitates continuous integration
  • We have several ways to interact with OpenShift: web application, CLI or API REST

Docker

“Build, Ship and Run. Any application, Anywhere”
  • Docker is a lightweight virtualization system that allows us to work with application containers.
  • The containers are self-sufficient (although they can depend on other containers), requiring nothing more than the image of the container for the services it offers to work.
  • Multiple containers share the same core, but each container can be restricted to using only a defined amount of resources such as CPU, memory and I/O.

Kubernetes

  • Kubernetes is an open source system that allows us to automatically deploy, scale and manage application containers.
  • It manages a set of servers where the containers are executed.
  • The real applications are formed by several containers, K8S offers us mechanisms to manage them: implementation, service assurance, error tolerance, scalability, continuous updates, integration and automatic deployments, load balancing, monitoring, ...

OpenShift

  • OKD (origin): Open source distribution that can be used in your own infrastructure.
  • OpenShift Online: Available on Red Hat's public cloud. Two plans: Free and Pro.
  • OpenShift Dedicated: Single-tenant, high-availability Kubernetes clusters in the public cloud
  • OpenShift Container Platform: A Openshift cluster in your own infrastructure managed by Red Hat.
