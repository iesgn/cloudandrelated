Applications evolution

The way the applications are developed and deployed is changing, as well as the applications themselves.

The typical application some years ago was running locally on a PC, however today most of the applications run on a remote web server and are used trough a web browser or a mobile app.

If the applications run mostly on remote servers, the number of users can vary abruptly, so it's critical to adapt the hardware resources to tohse variations.

The applications must SCALE

Scale up vs scale out

  • Scale up or vertical scaling: Adding more resources to the system when needed
    • "Easy" to scale up
    • Limited to the resources available to one system
    • It can be difficult to scale down
    • The application doesn't need to be adapted
  • Scale out or horizontal scaling: Asociated to distributed architectures. More nodes are added when needed
    • Needs some automation to scale
    • Easy to increase or decrease used resources
    • "Unlimited resources"
    • The application needs to be adapted

Monolithic application

  • All the components executed in the same node
  • Scale up
  • Simple architecture
  • Interferences between components
  • The application is usually updated as a whole
  • Static infrastructure for years
  • Application is not fault tolerant

Distributed application

  • Ideally, it uses one component per node
  • Scale out
  • Complex architecture
  • Less interferences between components
  • Each component can be updated separately

SOA

  • Service-Oriented Architecture
  • Independent services are provided to the other components through a communication protocol over a network
  • XML, SOAP, WSDL, ...
  • Messaging queues to avoid persistent connections
  • It was an attempt to develop applications in a standard way
    • Related to corporate applications some years ago and not widely used

Cloud native

  • Emphasis on adapting infrastructure to demand
  • Massive use of elasticity: Dynamic infrastructure
  • Resilient applications: Fault tolerant applications run on standard hardware
  • Scale out
  • Automatization

Microservices

  • A non-standard evolution of SOA
  • There's no a formal definition
  • Radical implementation of services: One process per node (microservices)
  • Communication through RESTful APIs
  • Related to agility
