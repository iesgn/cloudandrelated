Document format

  • The files used by applications don't use binary formats (fortunately we're not in the 90's)
  • Plain text files are commonly used
  • Markup Languages are the standard today for document formats:
    • They can be read by both applications and humans
    • Easy to modify directly by humans
    • Easy to code applications that can read or write them
  • The question is "which markup language to choose?"

XML

  • eXtensible Markup Language
  • Well-formed documents are parseable
  • XML documents can be defined and validated strictly with schemas
  • Widely used some years ago
  • Extensively used in the web, and java and .net frameworks
  • It's a basic element on SOA
  • It was the language used in REST API initially
  • Not commonly used in new applications today
  • Some people prefer a simpler markup language. They argue that open and close tags shouldn't be necessary

JSON

  • JavaScript Object Notation
  • Simpler than XML
  • There's a project to define schemes and a validation framework to JSON: JSON Schema but it's not commonly used
  • Widely used in REST APIs

YAML

  • YAML Ain't Markup Language
  • Even simpler than JSON
  • It doesn't use closing tags or brackets
  • Used to conf files in many modern applications
  • Widely used in many applications related to cloud computing
